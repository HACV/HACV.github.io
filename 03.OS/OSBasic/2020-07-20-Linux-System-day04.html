        # 一、终端

        计算机中的终端： 是所有输入输出设备的总称。 键盘，鼠标，摄像头 屏幕，打印机

        虚拟机那个黑框框。是终端，是虚拟终端，不是真实存在的 输入可以在这，输出也在这出，又输入有输出

        • 终端在我们系统的级别分，大致如下：
        • Alt + Ctrl + F1、F2、F3、F4、F5、F6 字符终端 pts (pseudo terminal slave) 指伪终端。
        • Alt + F7 图形终端
        • SSH、Telnet... 网络终端（比如用Xshell啥的，通过网络传输到主机上）

        init进程是1号进程，所有进程原先的祖先 0号进程是不会在操作系统中实际存在的。

        # 二、进程组（作业）

        （有的书上叫“作业”，重要！！）只不过在不同操作系统下，叫法不同而已。 作业这个概念，早先的时候，来源于BSD操作系统

        • BSD于1980年前后向Unix中增加的一个新特性。代表一个或多个进程的集合。每个进程都属于一个进程组。

        可以用kill，进行进程组ID取负号的方式，来杀死整个组内的进程。

        • 组长进程可以创建一个进程组

        # 三、会话（session）

        把一组进程组再重新编号，就变成了会话。。。（圈圈圆圆圈圈，禁止套娃2333）

        创建一个会话需要注意以下6点注意事项：

        1. 调用进程不能是进程组组长，该进程变成新会话首进程(session header)（简称，“会长”，滑稽）
        2. 该进程成为一个新进程组的组长进程。
        3. 需有root权限(ubuntu不需要)
        4. 新会话丢弃原有的控制终端，该会话没有控制终端（只在后台执行）
        5. 该调用进程是组长进程，则出错返回
        6. 建立新会话时，先调用fork, 父进程终止，子进程调用setsid

        要会话ID干啥？？主要是为了，守护进程用

        # 四、守护进程（重要）

        • Daemon(精灵)进程，是Linux中的后台服务进程，通常独立于控制终端并且周期性地执行某种任务或等待处理某些发生的事件。一般采用以d结尾的名字。
        • 比如，Linux中的vsftpd，搭建ftp服务器的，安装+启动，，httpd，sshd,xinetd
        • 只要是后台的，有没有终端？？没有。
        • 这个，守护进程，说白了，就是一个后台系统服务器。

        创建守护进程，最关键的一步是调用setsid函数创建一个新的Session，并成为Session Leader。

        注意，守护进程不受，不受用户登录、注销的影响，当然，机器重启就另说了,但是厉害的守护进程，比如vsftpd和mysql等，你关机再开机，它也跟着启动 当然，你也可以写这样的守护进程，比

        为什么，要用会话呢？ 因为守护进程都不需要控制终端。

        # 1）创建守护进程模型

        1. 创建子进程，父进程退出 所有工作在子进程中进行形式上脱离了控制终端
        2. 在子进程中创建新会话（主要，为了丢弃终端） 　　 setsid()函数 　　 使子进程完全独立出来，脱离控制
        3. 改变当前目录为根目录 　　 chdir()函数 　　 防止占用可卸载的文件系统（比如，U盘） 　　 也可以换成其它路径
        4. 重设文件权限掩码 　　 umask()函数 　　 防止继承的文件创建屏蔽字拒绝某些权限 　　 增加守护进程灵活性
        5. 关闭文件描述符 　　 继承的打开文件不会用到，浪费系统资源，无法卸载
        6. 开始执行守护进程核心工作（守护进程主逻辑）
        7. 守护进程退出处理程序模型 （几乎不会用，毕竟守护进程一般不退出）

        # 2）写一个守护进程

        #include <stdio.h>
#include <stdlib.h>
#include <fcntl.h>
#include <sys/stat.h>
#include <unistd.h>

void mydaemond(void)
{
	//1.创建子进程
    pid_t pid = fork();
    if (pid > 0) {
        exit(1);
    }


    //2.在子进程中创建新会话
    setsid();

    //3.	改变当前目录为根目录
    int ret = chdir("/home/itcast/");
    if (ret == -1) {
        perror("chdir error");  // chdir error no such diractroy or file
        exit(1);
    }

    //4.重设文件权限掩码
    umask(0022);

    //5.关闭文件描述符
    //close(fd[0]);  //stdin
    close(STDIN_FILENO);

    open("/dev/null", O_RDWR);
    dup2(0, STDOUT_FILENO);
    dup2(0, STDERR_FILENO);
}


int main(void)
{
    mydaemond();


    //6.开始执行守护进程核心工作（守护进程主逻辑）
    while (1) {
        
        //7.退出的话，需要在这注册一个信号捕捉
    }

    return 0;
}
