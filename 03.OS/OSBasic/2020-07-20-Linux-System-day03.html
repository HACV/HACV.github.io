        实际上计算机里面所看到的所有的东西，在生活当中都会能够找到他的影子。和印证、

        # 信号的特征

        1.简单2.不能携带大量信息3.满足某个特设条件才发送。 所以，信号不能带个结构体过去。

        信号的原理和时钟中断的原理几乎是一样的，只不过时钟中断时通过硬件来完成的。 信号的话，是通过软件手段俩实现的。

        有的参考书，也称信号为软中断

        也就是借助软件手段实现的中断机制。 信号的特质：由于信号是通过软件方法实现，其实现手段导致信号有很强的延时性（这个说法是针对CPU说的）。但对于用户来说，这个延迟时间非常短，不易察觉。

        每个进程收到的所有信号，都是由内核负责发送的 内核处理。 所以严谨的讲，进程A给进程B发信号，这样的说法，应该说是A驱使内核给B发信号。

        # 二、如何产生信号？ 产生信号 5 种方式

        # 与信号相关的事件和状态

        产生信号 5 种方式：

        1. 按键产生，如：Ctrl+c、Ctrl+z、Ctrl+\ （比如Ctrl+Z 会导致，这个按键会导致内核向你当前执行的进程发送一个信号过去，这个信号会把进程杀死）
        2. 系统调用产生，如：kill、raise、abort
        3. 软件条件产生，如：定时器alarm
        4. 硬件异常产生，如：非法访问内存(段错误)、除0(浮点数例外)、内存对齐出错(总线错误)（这个总线错误，比如咋们在mmap建立映射区的时候出现的总线错误）
        5. 命令产生，如：kill命令，实际上其他命令也有，但一般研究这

        # 关于信号的状态和一些常用的名词。

        比如Ctrl+Z ，马上信号产生了-》马上递送-》递达 一般我们认为，从信号产生到最后递达，我们认为是瞬时的。可以说时间忽略不计。

        但是，信号产生之后，可能由于某种原因被阻塞(屏蔽)，这样就不能递达了，我们称之为“阻塞态”。Linux内核当中对这个阻塞态有一个描述的机制，这个描述他的机制对应的是一个集合：这个集合叫做“阻塞信号集”，还有另外一个名字，叫做“信号屏蔽字” 用此来描述信号的阻塞状态。

        与“信号屏蔽字”相呼应的，有另外一个集合，这个集合叫做：“未决信号集”.所谓未决，也就是这个信号没有被处理掉（注意理解这个，这个和前面）。所以说，他处于一个未决状态。也是在信号产生和递达之间，是**转瞬即逝（注意这个词语，虽然想吐槽，那设计这个干啥。。）**的

        递达：递送并且到达进程。 未决：产生和递达之间的状态。主要由于阻塞(屏蔽)导致该状态。

        # PCB结构体中，有2个非常重要的成员：

        1）未决信号集-----为一个集合 2）信号屏蔽字（阻塞信号集）-----为一个集合 关系：“阻塞信号集”，影响（决定）“未决信号集” 当然：决定的前提是，信号要产生。 比如：我的某某进程，提前阻塞某编号的信号，0-》1 但是这个时候，那个编号的信号本来就没产生，他的未决是0，那么“阻塞信号集”就没影响。

        比如： 一个信号，要是不正常的，未决，那么未决信号集对应的编号从0-》1，然后处理，那么马上要从1-》0

        而，阻塞信号集，要是信号被阻塞，那么对应的编号从0-》1，而，阻塞过程，未决信号集对应的编号的0和1翻转，不能马上进行，所以，阻塞信号集，影响未决信号集

        # 信号的处理方式: （假设没有前面那些奇奇怪怪的东西）

        1. 执行默认动作
        2. 忽略(丢弃) （注意，这个是信号的处理方式，并不是说没有处理）
        3. 捕捉(调用户处理函数)

        # 默认动作：

        Term：终止进程 Ign： 忽略信号 (默认即时对该种信号忽略操作) Core：终止进程，生成Core文件。(查验进程死亡原因， 用于gdb调试) 上面的core，比如，我有次编程出现段错误啥的，然后，我们发现下面显示，啥啥啥给你弄到了那里转储 也就是这，core文件就是来提供给你用来检查进程死亡原因的。主要应用于GDB调试， 比如gdb调试段错误

        -g gdb a.out run
        1

        这样run到的停止的位置，就是出现段错误的位置，但是为何呢？？？ 实际上，实际上就是因为，a.out的程序在被段错误杀死的同时，产生了一个core文件，这个歌文件专门用于内核调试，gdb可以把他拿出来。

        Stop：停止（暂停）进程 Cont：继续运行进程

        为了防止黑客，修改了很多信号的处理方式为忽略 那样，病毒，我们无法通过kill进行杀死， 所以，Linux内核保留了两个，能够在必要的时候，将进程切实有效的杀死的信号。不允许忽略和捕捉。 也就是特殊信号9和19号

        # 五、信号捕捉

        signal函数 注册一个信号捕捉函数： typedef void (*sighandler_t)(int); sighandler_t signal(int signum, sighandler_t handler); 该函数由ANSI定义，由于历史原因在不同版本的Unix和不同版本的Linux中可能有不同的行为。因此应该尽量避免使用它，取而代之使用sigaction函数。

	printf("hello world\n");

	it.it_value.tv_usec = 0;

	it.it_interval.tv_sec = 3;
	it.it_interval.tv_usec = 0;

		perror("setitimer error");
		return -1;
	}

	while(1);

	return 0;
}
