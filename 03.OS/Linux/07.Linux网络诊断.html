        # （1）虚拟网卡VMnet1和Vmnet8

        装完虚拟机之后，会出现两块虚拟网卡：

        VMnet1和Vmnet8（如上图）

        你的虚拟机会利用真实网卡，和你当前的这个真实计算机（物理机）进行通信

        好处：配置简单，同时，你的虚拟机设置和你的真实机同一个网段的IP地址，他就可以在

        虚拟机和真实机之间，直接通信。而且，还有好处，不光可以和你的真实机通信，而且，局域网类，如果还有同网段的其他计算机，他也可以直接访问。

        坏处：他需要占用你的网段的一个IP，如果你是在教室里面，可能很多同学同时学习，这个时候就会经常出现IP地址冲突。

        （学完计算机网络，私网和公网，看这个真是清晰）

        # （2）NAT或者Host only

        如果选择NAT。你的虚拟机是通过VMnet8这块假网卡，和你的真实机进行通信。

        如果选择，Hostonly，你的虚拟机是通过VMnet1这块假网卡，和你的真实机进行通信。（记忆方法，only one）

        他和桥接的区别：桥接用的真实网卡，所以能和局域网类，同一网段的真实机器通信（虚拟机中的Linux就和一台真正的电脑（真正的一台独立的计算机）是一样的）

        要是选择NAT或Hostonly，那么你的虚拟机，就只能和你的真实机通信。和你的局域网的其他计算机是不能进行通信了！！！

        好处，显然就是不用占用真实网段的IP地址。（感觉，就像是现实生活中的公网不够，划分子网，用私网的既视感，，，，应该差不多）

        NAT和Hostonly的区别

        Hostonly：仅能和我的本机通信。虚拟机，只能和我这台真实机通信

        NAT：你的虚拟机和这台真实机通信的基础上！！如果，你的真实机也能访问互联网，那么你学虚拟机，也能正确的访问互联网。

        演示，没有插网线，选择Hostonly：让真实机和虚拟机通信。
