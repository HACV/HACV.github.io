    # 三、Linux系统安装-系统分区

    • 编号2.2 Linux系统安装-系统分区

    # （1）概念

    • 内容比较抽象
      『系统分区——也叫磁盘分区』
      字面意思就是，把大的硬盘，按照我的需求，分成几个小硬盘。
      整个大的硬盘中查找数据：是很大的工程量。
      分成各个小硬盘：这样，就很好的查找了。
      比如，C盘放操作系统，D盘放应用软件，E盘放娱乐资料等
      进行了分区，用户进行数据读取效率会大大增加。

    > 磁盘分区

    磁盘分区是使用分区编辑器（partition editor）在磁盘上划分几个逻辑部分。碟片一旦划分成数个分区( Partition)，不同类的目录与文件可以存储进不同的分区。

    分区是有一定类型的，最早的时候，只有主分区，这一种分区。

    但是主分区有限制，他最多只能有4个。

    当我的硬盘越来越大，4个主分区，可能就不够使用。

    这个时候，我们就必须把主分区当中拿出来一个分区，把他化为扩展分区。

    注意：扩展分区，对于一块硬盘（似乎指的是物理上的一个）来讲，只能有一个！！！

    而且，主分区+扩展分区，最多只能4个。（也就是扩展分区和主分区要一起计算）

    他们，两种类型的分区，加起来，最多只能4个

    扩展分区：本身也不能写入数据，也不能格式化，他唯一的作用就是用来包含逻辑分区。

    逻辑分区，可以正常的进行写入数据和格式化。（这句话，怎么和前面有点自相矛盾的感觉，但是后面知晓，扩展分区中包含了小的一个个逻辑分区）

    那么。什么是逻辑分区？？？

    扩展分区，就是，在这个分区下面继续分小格子。（那个衣柜的比喻）

    注意：逻辑分区，可以正常的写入数据和进行格式化。这个限制，不是Linux限制，而是我们的硬盘限制。只要硬盘的结构不发生变化，这种限制，依然都会存在。

    # （2）硬盘正确的分区后

    Q：如果硬盘正确的分区那么我们就可以，写入数据了吗？？

    A：不是,我们的硬盘，还需要经过格式化之后，才能写入数据。

    注意，我们说的格式化，指的是高级格式化！！！

    也就是说，还有低级格式化。

    但是低级格式化，是硬盘的操作，不是我们操作系统当中的操作。不在我们的授课范围之内。

    格式化（高级格式化）其实是为了，写入我们的文件系统！！

    在我们的Windows当中可以识别的文件系统有（FAT16，FAT32，NTFS）

    在Linux当中，可以识别的文件系统有（EXT2，EXT3，EXT4）

    CentOS6.3当中，使用的文件系统，默认的是EXT4。

    格式化（高级格式化）的目的，就是为了在硬盘当中，写入inich(不知道，他说的啥)

    比如，有了柜子就能放入衣服了吗？？其实，我们的柜子还需要写入相应的隔断。

    比如，某某某，放挂起来的。

    格式化（）大家就可以想象成为，把柜子，打入隔断。

    只是这个隔断，他必须遵循一定的规则。

    在我们的Linux当中，默认的文件系统，EXT4，他进入格式化的时候，他需要把我们的柜子变成一个一个等大小的数据块。这个数据块，默认的标准大小是4KB。

    也就是说每4KB空间，放成一个小隔断。那假设我有一个数据是10KB

    那么，一个隔断放不下，就会进行拆分，把他放到3个柜子当中。

    当然，最后最后一个隔断中只放了2KB数据。剩余的2KB的空闲空间，是不能再被其他文件

    占用的。

    当然，这只是为了示意图的方便（什么鬼话？）。其实，我们的硬盘啊，他的柜子，并没有把数据块保存到同一个位置，也就是说，如果保存在3个数据块当中，也可能在这里保存4KB，在其他地方保存4KB。。。

    所以，在我们Windows当中，有个工具，叫做磁盘碎片整理。他的原理就是，想把保存文件的不同数据块，尽量的把他放在一起。这样的话，更加利于数据的读取。

    那么，柜子打隔断，或者说（高级格式化）

    最主要的目的，就是把我们的文件系统写入。

    写入文件系统当中，最主要的工作，就是按照我的文件系统的规则，把你的硬盘分割成等大小的数据块。我们把这个数据块，给他起了一个名字，把他叫做Block。

    其实Block就是英文当中数据块的意思。。

    那么，写入了数据块，格式化就结束了吗？

    并没有，他还要做一些规则：

    比如说，我的文件，放在这个数据块当中，当我用户要想读取这个文件的时候，我怎么知道，我文件当中的数据是保存在哪几个数据块当中的呢？？？这个时候，就需要利用到文件的分区表。这个时候，大家可以这么想象：我在柜门上贴一个纸条，纸条上写入，我的某一件衣服放入了哪一个隔断，当我需要找这个衣服的时候，每个衣服都有一个编号。我通过这个表格的编号，查找到这个表格的这个衣服的这个数据条。

    然后，就能看到，我要找的衣服是放在，某个隔断，我就可以直接打开隔断，拿出我的衣服。

    还记得吗？？我说，每个文件，都应该有他的一个编号。

    我们找文件的时候，实际上是靠这个编号，来找到这个文件的相关条款。通过这个相关条款，就可以知道，这个文件放在了哪几个数据块当中，那么这个编号，我们也有一个称呼：

    大家其实可以把这个编号想象成id号。每个文件都有一个。

    理论上来说，他是唯一的。

    我们这个编号，把他称作，i节点号。（也叫做inode号）

    这就是分区最主要的工作，这就是格式化（高级）最主要的工作。

    大硬盘-》小硬盘-》写入文件系统（分成数据块）-》建立inode列表。

    当我们查找文件的时候，是通过我们的inode号，来找到这个文件的条款。

    从而知道，这个文件是保存在哪几个数据块当中，从而打开这个几个数据块

    拿出数据，拼凑成我们的完整文件。这就是格式化的作用。

    如上，我们硬盘有了文件名，还要给我们的分区也要分配文件名。

    如上是第一个分区，要是第二个分区，则要加2

    分区的设备文件名，就是在硬盘的设备文件名之后加入数字1，2,3,4啥的

    显然，不管你如何分区，这个5都代表，第一个逻辑分区。

    # （6）有了分区就能正常使用了吗？

    还少最后一步，挂载，（分配盘符的说法是Windows当中的称呼）

    我们把这个“盘符”叫做挂载点。

    换句话说，“盘符”是Windows当中的说法，Linux当中，应该说是挂载点。

    也就是说，我们的分区有了。我们在正常使用之前，还必须给我们的分区指定一个挂载点！！！、

    区别：

    Windows中是用C,D,E,F啥的来作为盘符

    Linux下，是用空的目录名称，来挂载（理论上，任何一个目录都可以作为盘符，但是是理论上，有些是不能的）

    ​

    对于，Linux来说，必须作为分区的有2个。

    1） 根分区，要是给根不分区，我们剩余的数据很多数据就没有地方可写。所以根必须要和一个分区。

    2） Swap分区，也叫交换分区。这个概念很难理解。。。

    可以这么想，所有的分区，其实就可以理解为虚拟内存。

    也就是说，当我的真正的内存不够用的时候，可以拿这部分交换分区的硬盘空间来，当内存来用。理论上来讲，我的交换分区，应该分，内存的2倍，也就是说，内存如果是1GB。那我的交换分区应该是2GB

    但是， 最大不超过2GB，原因很简单，虚拟分区，到底不是真正的内存！！！！给他再大，也不能取代内存。。

    当他达到2GB，如果再给他，只会，占用我们更多的硬盘空间。

    上面两个是必须分区的！！如果，不给他进行分区，我们Linux 是不能安装。只要有这两个分区，Linux就可以正常使用。

    但是我们推荐，把boot目录，单独分区。

    因为，很简单，任何一个操作系统，要想正常启动，都必须有一定的空余空间

    那，如果我们没有给boot单独分区，我所有的数据都放在根分区下，这个时候，万一，我把分区写满了，那么我的Linux就可能启动不起来了。。。

    为了解决这个问题，就可以把boot单独分区。

    Boot是用来保存，启动的时候的数据。Boot分区，不用太大，200MB足够。

    （如上，虽然这不是必须的，但是一般的Linux服务器当中，都会将这个单独分区。。）

    # （8）总结

    • 1、分区:把大硬盘分为小的逻辑分区

    • 2、格式化:写入文件系统，当然，在写入文件系统的同时，会附带的把硬盘当中的数据进行清空。（所以，这也是大家思维惯性的来源）

    • 3、分区设备文件名（自动识别）:给每个分区定义设备文件名

    • 4、挂载:给每个分区分配挂载点（必须是目录，而且必须是空目录）

    上面这样后，我们的分区才能正常的使用

    # （3）服务器注意事项

    1、远程服务器不允许关机，只能重启

    2、重启时应该关闭服务（因为服务器给很多人服务，你要是不关服务，直接管，可能会把硬件烧掉等。。）

    3、不要在服务器访问高峰运行高负载命令（这个，出问题的可能性会增加）

    比如，在Windows下执行某些操作，会非常卡。

    比如，大数据量的复制，全盘扫描（比如，杀毒）

    查找。大数据量的压缩和解压缩（十几二十G的文件）

    这些就是的。

    4、正常网站，3-5这个点，一般访问人数最少，可以做一些备份，什么的高负载操作。

    那么问题Q:我每天5点起来备份？？

    肯定不行，执行命令，自动在指定的时间执行备份。

    远程配置防火墙时不要把自己踢出服务器（我们在配置防火墙的时候，可以设置每5分钟，把防火墙都清空一次，测完再去掉，这样至少保证，自己可以不要大老远的去那些地方。。)

    5、防火墙：过滤，或者说是门岗。过滤IP，端口号，MAC，符合规则请进来。

    （但是病毒可以伪装啊——

    防火墙不等于杀毒软件。互相不能替代。

    有点像撒网捕鱼的网，网眼（像规则）

    比如，22点ssh的端口，你要是自己关了，那你远程管理工具，不要玩了。。。

    指定合理的密码规范并定期更新

    6、合理分配权限（权限给得恰到好处，越少越好，够用就好）（系统启动服务，越少越好）

    干嘛要分配权限？？

    那是你从个人机的角度，一辈子就他了。

    但是服务器不同。

    7、定期备份重要数据和日志（非常非常重要，比如数据丢失就麻烦了。）

    比如，3，4年的心血，要是没有备份，可能就完了。。。

    备份原则（不要把鸡蛋放到同一个篮子里面）

    比如，98年发洪水。
