      # （1）rpm包命名规则

      httpd-2.2.15-15.e16.centos.1.i686.rpm
httpd	软件包名
2.2.15	软件版本
15		软件发布的次数
e16.centos	适合的Linux平台『操作系统版本』
i686	适合的硬件平台『CPU架构，比如32和64位』
rpm		rpm包扩展名

      『如果是x64，他指代的是64位的软件安装包，只能安装在64位的CPU上』

      注意：这样的，只是『大体』的命名规则

      • 3、还有部分安装包，没有写，适合哪个Linux类型版本，那就指代，任何Linux版本都可以安装

      # 二、Linux学习和Windows的区别

      • 编号：3.1 给初学者的建议-注意事项

      注意：

      Windows和Linux是相互独立的操作系统，我们不能完全将Windows的思维全弄过来，有的Windows使用习惯，不要带到Linux下来

      # （1）区别1

      • 1） Linux严格区分大小写（不仅命令区分大小写，而且文件名称也区分大小写，Linux的命令的选项也是区分的大小写，用户名也区分，配置文件中的选项也是严格区分的。）
      • 注意：Linux的命令是没有大小的，全小写，但是，比如环境变量，习惯性的作为大写。
      • 2） Windows则不是那么（命令不区分大小写，文件命名也不区分大小，（比如，ABC.txt和abc.txt就不能放在同一个目录下，不然，他会让你改为ABC（2）.txt）
      比如，Linux下ls可以，但LS就不行了
Windows下DIR可以，dir也可以！！！

      # （2）区别2

      • （1）Linux中所有内容以文件形式保存，包括硬件

        ➢硬盘文件是/dev/sd[a-p]

        ➢光盘文件是/dev/sr0等

      • 其实，不光是硬盘，我们可以这么说，Linux当中，如果你想要这个内容，他能够永久生效。他归根结底都要写入文件，不管是IP地址**，网关，用户，等等，这样的一些信息。**他都要写入文件，如果没有写入文件，仅仅是写入了内存（比如命令行的修改）当中，那是临时生效，一旦重启就没有了。

      • （2）Windows下，硬件就是硬件，是通过『设备管理器』来管理硬件，但是我们的Linux当中，所有东西都是文件。包括，硬件，我们是通过管理和操作这个文件来操作对应的硬件。

      # 三、系统分区

      • 编号2.2 Linux系统安装-系统分区

      内容比较抽象
      『系统分区——也叫磁盘分区』
      字面意思就是，把大的硬盘，按照我的需求，分成几个小硬盘。
      整个大的硬盘中查找数据：是很大的工程量。
      分成各个小硬盘：这样，就很好的查找了。
      比如，C盘放操作系统，D盘放应用软件，E盘放娱乐资料等
      进行了分区，用户进行数据读取效率会大大增加。

      # （6）总结

      • 1、分区:把大硬盘分为小的逻辑分区

      • 2、格式化:写入文件系统，当然，在写入文件系统的同时，会附带的把硬盘当中的数据进行清空。（所以，这也是大家思维惯性的来源）

      • 3、分区设备文件名（自动识别）:给每个分区定义设备文件名

      • 4、挂载:给每个分区分配挂载点（必须是目录，而且必须是空目录）

      上面这样后，我们的分区才能正常的使用
