    # 二、Linux学习和Windows的区别

    • 编号：3.1 给初学者的建议-注意事项

    注意：

    Windows和Linux是相互独立的操作系统，我们不能完全将Windows的思维全弄过来，有的Windows使用习惯，不要带到Linux下来

    # （1）区别1

    • 1） Linux严格区分大小写（不仅命令区分大小写，而且文件名称也区分大小写，Linux的命令的选项也是区分的大小写，用户名也区分，配置文件中的选项也是严格区分的。）
    • 注意：Linux的命令是没有大小的，全小写，但是，比如环境变量，习惯性的作为大写。
    • 2） Windows则不是那么（命令不区分大小写，文件命名也不区分大小，（比如，ABC.txt和abc.txt就不能放在同一个目录下，不然，他会让你改为ABC（2）.txt）
    比如，Linux下ls可以，但LS就不行了
Windows下DIR可以，dir也可以！！！

    # （2）区别2

    • （1）Linux中所有内容以文件形式保存，包括硬件

      ➢硬盘文件是/dev/sd[a-p]

      ➢光盘文件是/dev/sr0等

    • 其实，不光是硬盘，我们可以这么说，Linux当中，如果你想要这个内容，他能够永久生效。他归根结底都要写入文件，不管是IP地址**，网关，用户，等等，这样的一些信息。**他都要写入文件，如果没有写入文件，仅仅是写入了内存（比如命令行的修改）当中，那是临时生效，一旦重启就没有了。

    • （2）Windows下，硬件就是硬件，是通过『设备管理器』来管理硬件，但是我们的Linux当中，所有东西都是文件。包括，硬件，我们是通过管理和操作这个文件来操作对应的硬件。

    # （3）区别3

    1） Linux不靠扩展名区分文件类型，那么是靠什么区分的呢？是靠文件权限。

    一些约定俗称的扩展名：

    配置文件.conf

    但是不是绝对的，比如MySQL的配置文件，是以.cnf结尾的，但是其实也是这个的缩写。

    比如，压缩文件。下载一个压缩包，要是啥扩展名都没写。

    我们Linux下能识别的压缩包格式比Windows还要多，有十几种。

    那我可能就要挨个的把十几个命令。全试一遍。

    当然，Linux是有命令可以来区分文件类型的，但是那个不直观。

    2）Windows则依靠，是依靠扩展名来区分文件类型

    此外，Windows还有好处，有图标便于区分。

    Linux下是不是所有文件都没有扩展名？？？

    Linux最开始就没有扩展名的概念。

    Windows最早的时候都要遵循8.3****规则。

    指的是：你的文件名，不应该超过8位，而且应该包括.3位，3位数的扩展名文件。

    比如说.doc，.ppt，.exe

    我们现在的Windows当中，也已经突破了这种限制。8.3规则，一般是接触电脑比较久的才晓得。

    现在文件名，早就突破了8位。而且，扩展名也有什么.docx

    突破了3位的扩展名。

    # 三、系统分区

    • 编号2.2 Linux系统安装-系统分区

    内容比较抽象
    『系统分区——也叫磁盘分区』
    字面意思就是，把大的硬盘，按照我的需求，分成几个小硬盘。
    整个大的硬盘中查找数据：是很大的工程量。
    分成各个小硬盘：这样，就很好的查找了。
    比如，C盘放操作系统，D盘放应用软件，E盘放娱乐资料等
    进行了分区，用户进行数据读取效率会大大增加。

    # （6）总结

    • 1、分区:把大硬盘分为小的逻辑分区

    • 2、格式化:写入文件系统，当然，在写入文件系统的同时，会附带的把硬盘当中的数据进行清空。（所以，这也是大家思维惯性的来源）

    • 3、分区设备文件名（自动识别）:给每个分区定义设备文件名

    • 4、挂载:给每个分区分配挂载点（必须是目录，而且必须是空目录）

    上面这样后，我们的分区才能正常的使用

    # （3）服务器注意事项

    1、远程服务器不允许关机，只能重启

    2、重启时应该关闭服务（因为服务器给很多人服务，你要是不关服务，直接管，可能会把硬件烧掉等。。）

    3、不要在服务器访问高峰运行高负载命令（这个，出问题的可能性会增加）

    比如，在Windows下执行某些操作，会非常卡。

    比如，大数据量的复制，全盘扫描（比如，杀毒）

    查找。大数据量的压缩和解压缩（十几二十G的文件）

    这些就是的。

    4、正常网站，3-5这个点，一般访问人数最少，可以做一些备份，什么的高负载操作。

    那么问题Q:我每天5点起来备份？？

    肯定不行，执行命令，自动在指定的时间执行备份。

    远程配置防火墙时不要把自己踢出服务器（我们在配置防火墙的时候，可以设置每5分钟，把防火墙都清空一次，测完再去掉，这样至少保证，自己可以不要大老远的去那些地方。。)

    5、防火墙：过滤，或者说是门岗。过滤IP，端口号，MAC，符合规则请进来。

    （但是病毒可以伪装啊——

    防火墙不等于杀毒软件。互相不能替代。

    有点像撒网捕鱼的网，网眼（像规则）

    比如，22点ssh的端口，你要是自己关了，那你远程管理工具，不要玩了。。。

    指定合理的密码规范并定期更新

    6、合理分配权限（权限给得恰到好处，越少越好，够用就好）（系统启动服务，越少越好）

    干嘛要分配权限？？

    那是你从个人机的角度，一辈子就他了。

    但是服务器不同。

    7、定期备份重要数据和日志（非常非常重要，比如数据丢失就麻烦了。）

    比如，3，4年的心血，要是没有备份，可能就完了。。。

    备份原则（不要把鸡蛋放到同一个篮子里面）

    比如，98年发洪水。
