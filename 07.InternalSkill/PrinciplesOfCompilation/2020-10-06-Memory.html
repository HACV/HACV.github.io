        # 内存对齐规则

        # 一、结构体内存对齐规则

        • 1.第一个成员在结构体变量偏移量为0 的地址处，也就是第一个成员必须从头开始。

        • 2.其他成员变量要对齐到某个数字（对齐数）的整数倍的地址处。对齐数为编译器默认的一个对齐数与该成员大小中的较小值（min（编译器默认对齐数，成员变量大小））。

        • Linux中gcc和g++编译器，32位默认对齐数是4字节对齐。64位默认字节对齐数是8字节对齐。（后续有证明，注意，参考的“脚本之家”那个有误）此外默认对齐数可以通过#pragma pack（）修改，但修改只能设置成1，2，4，8，16。

        • 3.结构体总大小为最大对齐数的整数倍（max（各个成员变量的由（规则2）获得的对齐数大小））。（每个成员变量都有自己的对齐数）

        • 4.如果嵌套结构体，嵌套的结构体对齐到自己的最大对齐数的整数倍处，结构体的整体大小就是所有最大对齐数（包含嵌套结构体的对齐数）的整数倍。

        • 5.可以借助，C语言头文件中#include<stddef.h>中函数offsetof观察现象

        # 二、内存对齐规则其他

        # 1、Linux下64位的默认对齐数是8字节证明

        编译条件：DevC++4.9.2 C语言文件，64位Release版本

        #include<stdio.h>
#include<stddef.h> 

struct A{
	char a;
	double b;
};

int main()
{
	printf("%d\n",sizeof(struct A));
	printf("%d\n",offsetof( struct A , a));
	printf("%d\n",offsetof( struct A , b));
	return 0;
}

        # 2、其他

        long 是由于历史原因导致的和int是一样，4个字节 short 是2个字节 结构体中union等，要是没有实例化，并且没有其他元素，那么不用算他占的大小 如果，仅仅只有它，需要算1个字节，毕竟如果不算，那就不好表述struct了

        64位gcc下输出

24
        1
        2

        原因：64gcc下对齐是8 union 没有初始化，但是struct中还有其他成员 我们就可以不用给它分配空间

        min（int , 8） 4字节对齐，int 占据前4个 min（short,8） 4字节对齐，short仅接着据4个 min（int , 8） 4字节对齐，int 占据前4个 min（char , 8） 1字节对齐，char 占据前1个 这个时候，我们发现，获得的是13个字节 接着，结构体总大小进行对齐 max（4,4,4,1）发现，最大对齐数是4 那么，我们知道是16
