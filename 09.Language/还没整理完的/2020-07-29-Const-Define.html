      # const和#define对比

      # 一、综述

      const的用途： 其实是C++里面，用来替换#define的一个方法，一个手段。

      C和C++中const和#define的区别：（？） 有相同的地方，也有不同的地方

      注意：数组下标不能用变量！！

      摘选自我写的另一篇博客的部分

      • 概述： Q：C语言中，const关键字限定一个变量为只读，但它是真正意义上的只读吗？ A：C语言中的并不是的！const虽说是constant的简写，是不变的意思。 但在C语言中，const并不是说它修饰常量，而是说它限定一个变量为只读。
      • C语言中，const关键字修饰的变量并非真正意义完完全全的只读！ （难怪叫常变量,因为C语言中const就没有把这个变量变成常量！） Tips：C++中则对const关键字进行了加强，使得真的变成了常量（详情见C++中const详解） 所以，我们常说C++中const修饰的变量，叫做const常量。

      综上，所以C语言中const修饰的变量，由于底层还是变量，所以不能作为数组下标 C++中const修饰的变量，由于在没有使用地址等情况，是常量，所以可以作为数组下标
